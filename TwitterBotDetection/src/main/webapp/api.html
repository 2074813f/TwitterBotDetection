Problem Statement


  • Users of the system should be able to provide a valid twitter userId, and receive a classification for the User.

  • Users of the system should be able to submit corrections to classifications for Users.

API Resources


classify(...)

Field Value
URI /rest/classify?userid=<…>
Method GET
Description Request a classification for the user with id given by the parameter "userid". Returns a classification object iff the id is a valid userid (i.e. the user exists and is accessible), else returns null.

Example:

http://localhost:8080/rest/classify?userid=791455969016442881

This requests a classification for the user with userid=791455969016442881, returning a populated response object if the user is successfully retrieved, or an HTTP error if not.

label(...)

Field Value
URI /rest/label?userid=<…>
Method POST
Description Provide a manual classification i.e. label for user with id given by the parameter "userid". Returns null on success.

Example:

http://localhost:8080/rest/label?userid=791455969016442881&label=human

This requests that the server recognize a manual classification for the user with userid=791455969016442881, as a human user.

Object Models


UserClassification
Attribute Description
userid:long The id of the user.
label:String The classification of the user, i.e. "human" or "bot"
features:Object Container for a number of "features" used in the classification of the user by the system.
